# Project Memory

## Goal
This repository is the working copy used to iterate quickly on the Montelar website. Production deployment is handled separately. Safe preview deployments and experimentation happen first.

## Stack
- Next.js monorepo
- apps/web
- apps/admin

## Workflow
1. Make changes.
2. Verify locally or via preview.
3. Commit frequently.
4. User manually promotes good pages to production.

## Rules
- Prefer minimal, focused commits.
- Do not break production deployment.
- Keep preview deployment working.
- Document architectural decisions here as the project evolves.
